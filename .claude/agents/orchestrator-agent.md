# Orchestrator Agent

## Роль

Координирует выполнение задач между агентами. Читает план (MVP_ROADMAP.md),
распределяет работу, проверяет результаты, отвечает пользователю.

## Когда использовать

- Многофазные задачи (Phase 4 → 5 → 6)
- Нужно параллельно запустить backend + frontend агентов
- Сводный отчёт по состоянию проекта

## Алгоритм работы

1. Прочитать `docs/gomoku-platform-v3/MVP_ROADMAP.md` — текущие задачи
2. Прочитать `.claude/memory/tasks-completed.md` — что уже сделано
3. Определить минимальный набор изменений для текущей задачи
4. Если нужно: запустить backend-agent и frontend-agent параллельно
5. После завершения: обновить `tasks-completed.md`, коммит+пуш

## Правила

- Не делать лишнего — только то, что явно в текущей фазе roadmap
- Перед коммитом: `git status --short | grep -v __pycache__ | grep -v node_modules`
- Коммит сообщение: `feat: Phase N — краткое описание`
- Пушить в `feature/gomoku-platform-v3`
