# Профиль пользователя — как работать вместе

## Технический бэкграунд

- Уверенно работает с: TypeScript/JavaScript, Python, Docker, Git
- Понимает C++ на уровне чтения и ревью, не пишет с нуля
- Опыт с ML: TensorFlow.js в продакшне, PyTorch на уровне использования
- Инфраструктура: Windows 11, WSL/bash, NVIDIA RTX 3060 Laptop, CUDA 12, WDDM
- Хорошо разбирается в архитектуре, думает системно

## Стиль взаимодействия

- Предпочитает **русский язык** в общении
- Одобряет планы явно: «да утверждаю», «переходи к Phase N» — это сигнал к выполнению
- Не нужно пересказывать что было сделано — он это видит
- Короткие ответы лучше длинных, если суть передаётся
- Ценит конкретику: числа, имена файлов, команды — не абстрактные описания

## Рабочий процесс

- Работает phase-by-phase: каждую фазу явно утверждает перед стартом
- Запускает обучение/тесты самостоятельно, просит диагностику по результату
- Предпочитает коммитить и пушить после завершения каждого этапа
- Документацию ценит — MD файлы с реальными данными (не шаблоны-заглушки)
- Предпочитает `make <target>` как единую точку запуска

## Чего избегать

- Не задавать лишних уточняющих вопросов если контекст есть в коде
- Не создавать абстракции «на вырост» — только то, что нужно сейчас
- Не добавлять комментарии к коду без просьбы
- Не делать backwards-compatibility заглушки при удалении кода
- Не использовать `git add -A` — только конкретные файлы
