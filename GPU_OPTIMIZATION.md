# GPU Оптимизация для максимальной загрузки

## Текущая проблема
GPU загружена только на ~20%, мощность ~18-19 Вт вместо ожидаемых ~80 Вт для RTX 3060 Laptop.

## Внесенные изменения

### 1. Увеличен batch size
- **Основное обучение**: 512 → **2048** (увеличено в 4 раза)
- **Дообучение**: 128 → **512** (увеличено в 4 раза)  
- **Фоновое дообучение**: 64 → **512** (увеличено в 8 раз)

### 2. Увеличен размер модели
- **dModel**: 128 → **256** (увеличено в 2 раза)
- **numLayers**: 4 → **6** (увеличено на 50%)
- **heads**: 4 → **8** (увеличено в 2 раза)

## Ожидаемые результаты
- Увеличение загрузки GPU с ~20% до 70-90%
- Увеличение потребления мощности с ~18-19 Вт до 60-80 Вт
- Более эффективное использование VRAM
- Ускорение обучения благодаря лучшей параллелизации

## Мониторинг GPU
Запустите скрипт для мониторинга мощности GPU:
```bash
docker exec tic-tac-toe-server bash -c "cd /app/server && node monitor_gpu_power.mjs"
```

## Проверка текущей загрузки
```bash
docker exec tic-tac-toe-server bash -c "nvidia-smi --query-gpu=name,power.draw,power.limit,utilization.gpu,utilization.memory,memory.used,memory.total --format=csv,noheader"
```

## Дополнительные рекомендации
1. Если GPU все еще не загружена полностью:
   - Увеличьте batch size до 4096
   - Используйте mixed precision training (FP16)
   - Увеличьте dModel до 512
   
2. Если не хватает памяти:
   - Уменьшите batch size до 1024
   - Используйте gradient accumulation
   - Уменьшите dModel до 192
