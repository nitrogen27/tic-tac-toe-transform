# Валидация настроек на клиенте

## ✅ Реализовано

### 1. Валидация настроек основного обучения
- **Количество эпох** (`mainTrainingEpochs`):
  - Диапазон: 1-10
  - Автоматическая коррекция при вводе
  - Проверка на целое число
  
- **Размер батча** (`mainTrainingBatchSize`):
  - Диапазон: 128-4096
  - Автоматическое округление до кратного 128
  - Проверка на целое число

### 2. Валидация настроек дообучения
- **Количество эпох** (`trainingEpochs`):
  - Диапазон: 1-10
  - Автоматическая коррекция при вводе
  - Проверка на целое число
  
- **Размер батча** (`incrementalBatchSize`):
  - Диапазон: 32-1024
  - Автоматическое округление до кратного 32
  - Проверка на целое число
  
- **Вариаций паттерна на ошибку** (`patternsPerError`):
  - Диапазон: 10-2000
  - Автоматическое округление до кратного 10
  - Проверка на целое число

### 3. Автоматическая коррекция значений
- При вводе значения вне диапазона - автоматически корректируется
- При вводе нецелого числа - округляется
- При вводе значения не кратного нужному шагу - округляется до ближайшего кратного

### 4. Валидация перед отправкой
- `startTrain()` проверяет настройки основного обучения перед отправкой
- `trainOnGames()` проверяет настройки дообучения перед отправкой
- Если валидация не прошла - показывается сообщение об ошибке

### 5. Отображение ошибок
- Ошибки валидации отображаются под каждым полем ввода
- Красный цвет для ошибок
- Сообщения на русском языке

## Использование

1. **Ввод значений**: Пользователь вводит значения в поля настроек
2. **Автоматическая коррекция**: При вводе значения автоматически корректируются
3. **Валидация**: При потере фокуса (`blur`) и при изменении (`input`) проверяются значения
4. **Отображение ошибок**: Если есть ошибки - они отображаются под полем
5. **Проверка перед отправкой**: Перед отправкой запроса на обучение проверяются все настройки

## Примеры валидации

### Валидные значения:
- Эпохи: 1, 2, 5, 10 ✅
- Batch size (основной): 128, 256, 512, 1024, 2048, 4096 ✅
- Batch size (дообучение): 32, 64, 128, 256, 512, 1024 ✅
- Patterns per error: 10, 100, 500, 1000, 2000 ✅

### Невалидные значения (автоматически корректируются):
- Эпохи: 0 → 1, 20 → 10, 2.5 → 3
- Batch size: 100 → 128, 5000 → 4096, 129 → 128
- Patterns: 5 → 10, 3000 → 2000, 105 → 100
