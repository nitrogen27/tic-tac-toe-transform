PyTorch training stack for the Gomoku Platform V3 policy-value ResNet.

## Architecture

```
src/trainer_lab/
├── config.py              # ModelConfig, TrainConfig, SelfPlayConfig (Pydantic)
├── data/
│   ├── encoder.py         # board dict → [6, 16, 16] float tensor (6 planes)
│   ├── augmentation.py    # D4 symmetry group — 8 board+policy transforms
│   └── dataset.py         # PositionDataset loading JSON position files
├── models/
│   ├── blocks.py          # ResBlock, SEBlock (channel-attention stub)
│   └── resnet.py          # PolicyValueResNet — dual-head, 8 ResBlocks

│   └── trainer.py         # Full loop: AMP, checkpoints, TensorBoard
├── export/
│   └── onnx_export.py     # ONNX export — fixed [B, 6, 16, 16] input, opset 17
├── evaluation/
│   └── eval_script.py     # Policy accuracy + value calibration vs engine
└── self_play/             # Phase-6 skeleton
    ├── player.py          # Model wrapper for game generation
    ├── replay_buffer.py   # Circular buffer with capacity limit
    └── pipeline.py        # generate → train → eval loop stub
```

## Model

**PolicyValueResNet** — ResNet tower with dual output heads:

- **Input**: `[B, 6, 16, 16]` — 6 planes padded to 16×16:
  - Plane 0: my stones
  - Plane 1: opponent stones
  - Plane 2: legal moves mask
  - Plane 3: last move
  - Plane 4: side to move (constant)
  - Plane 5: board size mask
- **Tower**: Conv2d → BN → ReLU → 8× ResBlock (128 filters)
- **Policy head**: Conv(1×1) → BN → Flatten → FC → `[B, 256]` logits
- **Value head**: Conv(1×1) → BN → FC(128) → FC(1) → tanh → `[B, 1]` ∈ [-1, 1]

## Train

```python
from torch.utils.data import DataLoader
from trainer_lab.data.dataset import PositionDataset
from trainer_lab.training.trainer import train

# Positions are JSON files with: board_size, board, current_player,
# last_move, policy (256 floats), value (float in [-1, 1])
dataset = PositionDataset("data/", augment=True)   # 8× via D4 symmetry
loader = DataLoader(dataset, batch_size=256, shuffle=True)
model = train(loader)
```

Or via Makefile from repo root:
```bash
make train
```

## Data Format

Each position JSON file contains a list of records:

```json
[
  {
    "board_size": 15,
    "board": [[0,0,...], ...],
    "current_player": 1,
    "last_move": [7, 7],
    "policy": [0.0, 0.0, ..., 0.95, ...],
    "value": 0.12
  }
]
```

Positions are generated by the C++ engine (`engine-core/`) via the FastAPI gateway.
