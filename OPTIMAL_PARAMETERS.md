# Оптимальные параметры обучения модели

## Рекомендуемые настройки

На основе тестов обучаемости модели были подобраны оптимальные параметры для обеспечения качественной игры без ошибок:

### Основное обучение (обучение с нуля)

```javascript
{
  epochs: 5,        // Количество эпох
  batchSize: 2048   // Размер батча
}
```

**Обоснование:**
- **5 эпох** обеспечивают достаточное качество обучения для игры без ошибок против minimax
- **batchSize 2048** - оптимальный баланс между скоростью обучения и использованием GPU памяти
- При тестировании модель показала 100% успешных игр (10/10 побед против minimax)

### Дообучение (инкрементальное обучение)

```javascript
{
  epochs: 1,        // Количество эпох (быстрое дообучение)
  batchSize: 256    // Размер батча
}
```

**Обоснование:**
- **1 эпоха** достаточна для быстрого усвоения новых паттернов
- **batchSize 256** оптимален для небольших наборов данных

## Архитектура модели

```javascript
{
  dModel: 192,      // Размерность модели
```

## Применение настроек

Параметры уже установлены в:
- `server/src/config.mjs` - конфигурация сервера
- `client/src/App.vue` - значения по умолчанию в UI
- `server/server.mjs` - дефолтные значения при отсутствии payload

## Рекомендации

1. **Для быстрого тестирования:** используйте 1-2 эпохи с batchSize 4096
2. **Для качественной игры:** используйте 5 эпох с batchSize 2048 (рекомендуется)
3. **Для максимального качества:** используйте 10 эпох с batchSize 1024 (медленнее, но лучше)
